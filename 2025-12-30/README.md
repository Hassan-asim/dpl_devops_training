## üìö Notes & Key Learnings
- **What AWS SSO / IAM Identity Center is:** A centralized service to manage SSO access to multiple AWS accounts and supported business applications.
- **Core concepts learned:** Identity source (built-in or external IdP), users & groups, permission sets (reusable permission policies assigned to accounts), account assignments, and session duration.
- **Integration:** Supports external identity providers via SAML 2.0 and can integrate with enterprise IdPs (Okta, Azure AD, etc.).
- **Permissions model:** Permission sets map to IAM roles in target accounts and simplify cross-account access management.
- **Observability & audit:** Actions and changes can be tracked using CloudTrail logs for auditing access and assignments.
